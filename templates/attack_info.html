Attack Type Information

Detailed explanation of each attack category

Normal Traffic

Risk: NONE

What it is: Legitimate network activity from authorized users and systems.

Characteristics:

  • Regular connection patterns
  • Expected data transfer volumes
  • Authorized protocols and services

Action Required: None - Continue monitoring

Real-world Example: Employee accessing company email, browsing approved websites

DoS (Denial of Service)

Risk: HIGH (8/10)

What it is: Attacker overwhelms system resources to make services unavailable.

Attack Examples:

  • Neptune: SYN flood attack
  • Smurf: ICMP flood using amplification
  • Back: Apache web server attack
  • Teardrop: Fragmented packet attack

Action Required:

  • Block source IP addresses immediately
  • Increase bandwidth capacity
  • Enable DDoS protection services
  • Alert security team

Real-world Impact: Website crashes, services become unavailable, revenue loss

Probe (Scanning)

Risk: MEDIUM (6/10)

What it is: Attacker scans network to gather information about vulnerabilities.

Attack Examples:

  • Nmap: Network mapping tool
  • Portsweep: Scanning multiple ports
  • Ipsweep: Scanning multiple IP addresses
  • Satan: Security analysis tool

Action Required:

  • Monitor source closely
  • Strengthen firewall rules
  • Log all scanning attempts
  • Investigate source identity

Real-world Impact: Reconnaissance for future attacks, vulnerability discovery

R2L (Remote to Local)

Risk: CRITICAL (9/10)

What it is: Unauthorized access attempt from a remote machine to local system.

Attack Examples:

  • Guess_passwd: Password guessing attack
  • FTP_write: Unauthorized FTP write access
  • Imap: Email server exploitation
  • Phf: CGI script vulnerability

Action Required:

  • Block access immediately
  • Force password resets
  • Check for data breach
  • Audit all access logs
  • Enable multi-factor authentication

Real-world Impact: Data theft, unauthorized access, credential compromise

U2R (User to Root)

Risk: CRITICAL (10/10)

What it is: Attacker escalates privileges from normal user to administrator/root.

Attack Examples:

  • Buffer_overflow: Memory corruption exploit
  • Rootkit: Malicious software for root access
  • Loadmodule: Kernel module loading exploit
  • Perl: Script-based privilege escalation

Action Required:

  • ISOLATE SYSTEM IMMEDIATELY
  • Disconnect from network
  • Full forensic investigation
  • Preserve evidence
  • System rebuild may be required
  • Contact incident response team

Real-world Impact: Complete system compromise, data destruction, backdoor installation
